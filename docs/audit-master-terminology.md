# Аудит упоминаний роли «мастер» в проекте Kircraft

**Цель:** Найти все вхождения слова «мастер» и связанные формулировки для последующей унификации клиентской терминологии.  
**Правило:** Внутренние имена (screen id, переменные, комментарии), не видимые пользователю, не считаются клиентской проблемой.

## 5. Итог

### 5.1 Места, которые нужно менять для клиентского интерфейса

При переходе на нейтральную клиентскую терминологию (без слова «мастер» в обращении к клиенту) целесообразно изменить:

1. **Кнопки контакта (единая формулировка):**  
   «Написать мастеру» / «Связаться с мастером» → одна нейтральная («Написать в магазин», «Связаться с нами», «Поддержка») в файле `components/telegram-bot.tsx` на экранах: главное меню, order-detail, order-cancelled-auto, post-order-clarification, post-order-rejected.

2. **Тексты экранов заказа в `components/telegram-bot.tsx`:**
   - post-order-clarification: «Мастер оставил комментарий…» → «Оставлен комментарий к заказу…»; «Комментарий мастера:» → «Комментарий к заказу» / «Сообщение от магазина».
   - post-order-rejected: «мастер не может принять» → «мы не можем принять».
   - post-order-confirmed: «Мастер подтвердил ваш заказ» → «Заказ подтверждён» / «Мы подтвердили ваш заказ».
   - post-order-payment-received: «Заказ передан мастеру…» → «Заказ принят в работу…».
   - order-detail (статус IN_PRODUCTION): «Мастер начал работу…» → «Работа над заказом начата.»

3. **Тексты после оформления заказа в `components/telegram-bot.tsx`:**
   - «Мы передали заказ мастеру на подтверждение» → «Заказ передан на подтверждение».
   - В модальном окне: «Мастер свяжется с вами в Telegram» → «Мы свяжемся с вами в Telegram».

4. **Опционально (бренд/контекст «о нас»):**  
   Роль в команде «Мастер», «О мастерской», «Интерьер мастерской», «Кожевенная мастерская» — менять только при желании убрать слово «мастерская» из клиентского контекста; иначе можно оставить.

### 5.2 Места, которые можно оставить как внутренние или демо

- **Screen id:** `master-order-paid`, `master-order-in-production` — не отображаются пользователю.
- **Переменная:** `demoMasterComment` — внутреннее имя.
- **Демо-меню:** подпись «Действия мастера» и кнопка «Новый заказ у мастера» — служебная навигация по сценариям; можно оставить или слегка нейтрализовать при общей унификации.
- **Документация** (`docs/post-order-flow-audit.md`, `docs/ux-cart-and-orders.md`) — обновить после изменений в UI, не считать источником клиентских текстов.

### 5.3 Краткая рекомендация по единой терминологии для клиента

- **Обращение от имени магазина:** использовать «мы» вместо «мастер» в сообщениях о подтверждении, оплате, связи и т.п.
- **Кнопка контакта:** одна формулировка для всех экранов, например «Связаться с нами» или «Написать в магазин», без слова «мастер».
- **Комментарии к заказу:** «Комментарий к заказу» или «Сообщение от магазина» вместо «Комментарий мастера».
- **Внутренние и демо-наименования** (screen id, демо-подписи для мастера) можно не менять; при рефакторинге — по желанию привести в соответствие с новой терминологией.

После правок — однократно пройти по разделам 1–3 этого отчёта и обновить документацию и спец-аудиты (в т.ч. `docs/ux-cart-and-orders.md`, `docs/post-order-flow-audit.md`).
